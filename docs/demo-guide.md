# Novel2Script Demo 演示指南

本文档用于录制 Novel2Script Demo 视频。建议视频控制在 3 到 5 分钟，重点展示从小说输入到 YAML 生成、编辑、校验和导出的完整闭环。

## 1. 开场介绍

讲解词参考：

大家好，这是 Novel2Script，一个面向小说作者的 AI 小说转剧本工具。它解决的问题是：小说作者在改编剧本时，需要手动整理章节、人物、地点、场景和对白，过程重复且耗时。

Novel2Script 可以将 3 个章节以上的小说文本转换为结构化剧本 YAML，让作者获得可编辑、可校验、可导出的剧本初稿。

## 2. 展示小说输入

操作：

1. 打开首页。
2. 点击“加载示例小说”。
3. 简要滚动 textarea，说明示例小说包含第一章、第二章、第三章。

讲解词参考：

这里可以粘贴作者自己的小说文本，也可以加载项目内置的原创示例小说。当前工具要求输入至少 3 个章节以上文本，这是为了保证 AI 有足够的故事结构进行改编。

## 3. 展示章节检测

操作：

1. 指向“当前输入字数”。
2. 指向“检测到章节数”。
3. 指向章节预览列表。

讲解词参考：

系统会自动检测章节标题，支持“第一章”“第1章”“章节一”“Chapter 1”等常见格式。检测结果会显示字数、章节数量和章节列表。只有检测到 3 个章节以上时，生成按钮才可以点击。

## 4. 展示 AI 生成

操作：

1. 点击“生成剧本 YAML”。
2. 展示 loading 状态。

讲解词参考：

点击生成后，前端会调用后端 `/api/generate` 接口。后端读取本地环境变量中的模型配置，再调用 OpenAI-compatible API。AI 会分析章节、人物、地点、场景和对白，并输出结构化 YAML。

## 5. 展示 YAML 结果

操作：

1. 等待生成完成。
2. 展示 YAML 编辑区。
3. 简要指出几个顶层字段。

讲解词参考：

生成结果不是一段普通文本，而是结构化 YAML。顶层包含 `script`、`chapters`、`characters`、`locations`、`scenes` 和 `metadata`。这让后续编辑、校验和导出都更稳定。

## 6. 展示 YAML 编辑

操作：

1. 在 YAML 编辑器里修改一个字段，例如 `script.title` 或 `metadata.notes`。
2. 保持 YAML 格式正确。

讲解词参考：

作者可以直接在页面中继续编辑 AI 生成的结果。这个项目的定位不是替代作者，而是提供一个可继续打磨的剧本初稿。

## 7. 展示 Schema 校验

操作：

1. 点击“校验 YAML”。
2. 展示校验通过状态。
3. 可选：删除 `script.title` 后再次校验，展示失败结果和具体 path。

讲解词参考：

Novel2Script 提供了 YAML Schema 校验。合法内容会显示校验通过；如果缺少必填字段或格式错误，页面会列出具体 path 和 message，方便作者定位问题。

## 8. 展示导出

操作：

1. 点击“导出 YAML”。
2. 展示浏览器下载 `novel2script-output.yaml`。

讲解词参考：

当作者编辑和校验完成后，可以一键导出当前编辑器里的内容。导出的不是初始生成结果，而是用户当前修改后的 YAML。

## 9. 展示 Schema 文档

操作：

1. 打开 `docs/yaml-schema.md`。
2. 展示顶层结构、字段说明表和设计取舍。

讲解词参考：

项目还提供完整的 Schema 设计文档，说明为什么使用 YAML、为什么保留 chapters、为什么独立 characters 和 locations，以及每个字段的校验规则。

## 10. 收尾

讲解词参考：

Novel2Script 满足赛题要求：支持 3 章以上小说输入，调用 AI 生成结构化剧本 YAML，并提供 Schema 文档、在线编辑、校验和导出能力。它的核心价值是让小说作者快速获得可编辑、可校验、可继续创作的剧本初稿。
