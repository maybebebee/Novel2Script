# Novel2Script Demo 演示指南

本文档用于录制 Novel2Script Demo 视频。建议视频控制在 3 到 5 分钟，重点展示从小说输入到中文剧本修改、YAML 生成、Schema 校验和导出的完整闭环。

## 1. 开场介绍

Novel2Script 是一个面向小说作者的 AI 小说转剧本工具。它可以将 3 个章节以上的小说文本改编为可修改的中文剧本稿，并进一步生成结构化 YAML。

## 2. 展示小说输入

操作步骤：

1. 打开首页。
2. 点击“加载示例小说”。
3. 展示 textarea 中的三章节小说文本。

讲解重点：

- 用户可以粘贴自己的小说。
- 当前工具要求至少 3 个章节。
- 示例小说为原创内容，便于评委快速体验。

## 3. 展示章节检测

操作步骤：

1. 指向“当前输入字数”。
2. 指向“检测到章节数”。
3. 指向章节预览列表。

讲解重点：

- 系统支持“第一章”“第1章”“章节一”“Chapter 1”等常见章节标题。
- 只有检测到 3 个章节以上时，生成按钮才可点击。

## 4. 生成可修改剧本

操作步骤：

1. 点击“生成可修改剧本”。
2. 等待生成完成。

讲解重点：

- 前端调用 `/api/generate`。
- 后端读取环境变量中的模型配置，调用 OpenAI-compatible API。
- AI 返回的是符合 Schema 的结构化剧本数据，但页面优先展示中文剧本稿，而不是直接展示 YAML。

## 5. 修改中文剧本稿

操作步骤：

1. 展示剧本标题和一句话梗概。
2. 展示人物与地点设定。
3. 展示每一场的中文剧本格式。
4. 修改某一场的画面、动作、台词、音效或情绪。

讲解重点：

- 修改页面向作者和编剧，尽量接近正常中文剧本阅读格式。
- 每场包含画面、动作、台词、音效、情绪、剧情节拍和转场。
- 这些修改会同步到底层结构化对象，后续可以生成 YAML。

## 6. 生成 YAML

操作步骤：

1. 修改剧本后点击“生成 YAML”。
2. 展示 YAML 数据层。
3. 指出顶层字段：`script`、`chapters`、`characters`、`locations`、`scenes`、`metadata`。

讲解重点：

- YAML 不是一开始强迫作者阅读，而是在作者确认剧本稿后生成。
- YAML 数据可用于校验、导出和后续工具处理。
- 如果手动修改 YAML，也可以同步回中文剧本修改页。

## 7. 展示 Schema 校验

操作步骤：

1. 点击“校验 YAML”。
2. 展示校验通过状态。
3. 可选：删除一个必填字段后再次校验，展示失败 path。

讲解重点：

- 校验结果会展示具体 path 和 message。
- 这能帮助作者定位结构问题。

## 8. 展示导出

操作步骤：

1. 点击“导出 YAML”。
2. 展示浏览器下载 `novel2script-output.yaml`。

讲解重点：

- 导出的是用户修改后的最新 YAML。

## 9. 收尾

Novel2Script 满足赛题要求：支持 3 章以上小说输入，调用 AI 生成结构化剧本数据，并提供中文剧本编辑、Schema 文档、在线校验和导出能力。核心价值是让小说作者快速获得可继续创作的剧本初稿。
